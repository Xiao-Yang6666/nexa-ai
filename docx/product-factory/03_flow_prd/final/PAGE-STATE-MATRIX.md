# PAGE-STATE-MATRIX — 页面状态矩阵（S4/S6 状态覆盖权威输入）

> 项目：基于 new-api 的 AI API 网关 SaaS（RoutifyAPI）。
> 本文件遍历 14 个 `flow/FL-*.md` 中**每张图后的「屏幕状态清单」**，汇总成「页面/组件 × 状态」矩阵。
> 列定义：**默认态** / **loading 态** / **空态** / **成功态** / **失败态** / **权限拦截态** / **其他特殊态**（状态机态、内部态、降级态、二次确认等）。
> 本矩阵是 **S4 信息架构 / S6 原型状态覆盖的权威输入**：原型每个页面/组件必须覆盖此处列出的状态。
> 说明：系统内部链路（选渠/CAS/亲和等无屏幕渲染）以「内部态」标注于其他特殊态列，供后端/调试视图参考，不强制原型可视化。

---

## 矩阵图例

- `—` 表示该列对此页面无对应状态。
- 单元格内多个状态用 `；` 分隔。
- 来源标注 FL 文件与场景号（如 `FL-account AC-1`）。

## 汇总统计

| 维度 | 数量 |
|---|---|
| 覆盖页面/组件（矩阵行） | **63** |
| 来源场景（FL 场景 + F-4037） | **82**（81 个 FL 场景 + 1 个 F-4037 通知配置） |
| 矩阵分区（A~N） | 14 区，对应 14 个 FL 分片 + 公开站点合并区 |
| 状态条目（去重后跨 7 列累计的不同状态描述） | **约 290+ 个状态单元**（含默认/loading/空/成功/失败/权限拦截/特殊态七类） |

### 状态分布（七列覆盖度，按页面行计）

| 状态列 | 有此列状态的页面数 | 说明 |
|---|---:|---|
| 默认态 | 约 52 / 63 | 多数页面有明确默认/初始态；纯系统内部链路/外部 API 无默认 UI |
| loading 态 | 约 18 / 63 | 看板/数据获取流/流式输出/异步轮询页面有 |
| 空态 | 约 16 / 63 | 列表/看板/排行/详情类页面有空态占位 |
| 成功态 | **63 / 63** | 每个页面均有至少一个成功/终态 |
| 失败态 | 约 55 / 63 | 绝大多数页面有失败态；纯展示页（协议/主题切换部分）失败态较少 |
| 权限拦截态 | 约 22 / 63 | 受 AdminAuth/RootAuth/UserAuth/C1/C3 保护的页面有 |
| 其他特殊态 | 约 48 / 63 | 状态机态、内部态、降级态、二次确认、旁路态等 |

> 结论：**成功态 100% 覆盖**；失败态、默认态、特殊态高覆盖；loading/空态集中于看板与数据获取流页面（符合预期，纯校验链/状态机页面无 loading）。本矩阵可直接驱动 S4 信息架构的页面清单与 S6 原型的状态分镜。
