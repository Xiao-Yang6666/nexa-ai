# COMPAT-BILLING-PRODUCT-DESIGN — 兼容层 + 计费体系 产品交互设计

> 角色：产品交互/信息架构设计（不写 HTML/CSS、不画像素级 UI；像素稿留后续原型阶段）。
> 最高权威：`COMPAT-BILLING-DECISIONS.md`（用户拍板业务规则）。本文所有交互严格服从 DECISIONS，并与
> `reviews/BILLING-MODEL-ARCHITECTURE.md` 的数据模型（PublicModel / ChannelModelCost / group_ratio / Log 利润字段 / 请求时序）逐一对齐。
> 复用现有原型体系（admin/console/web-public 三套外壳与导航），**明确每个新增点是「在现有页加 Tab/区块」还是「全新页」，不另起导航树**。

---

## 0. 设计总纲（把数据模型落成"人怎么操作"）

| 数据对象（架构） | 谁来操作 | 落在哪个端/页 | 交互形态 |
|---|---|---|---|
| **PublicModel**（对外模型 A：品质/售价/上下架） | 超管 | admin `models-admin` 现有「对外模型」Tab 增强 | 列表 + 抽屉编辑 + 品质拆分引导 |
| **PlatformModelMapping**（A→B 底仓映射，客户不可见 B） | 超管 | admin `models-admin` 对外模型抽屉内「底仓映射」区块 | 抽屉内 A→B 单选 + 渠道挂载预览 |
| **ChannelModelCost**（渠道×真实模型 B 成本倍率，手填） | 超管 | admin `models-admin` 新增「供应商成本」Tab | 二维表（渠道×B）行内编辑 |
| **多供应商兜底**（A→B 下挂哪些渠道、优先级/权重） | 超管 | 复用 admin `channels` 渠道路由 + 对外模型抽屉「兜底池」只读视图 | 衔接现有渠道界面，不重做路由 |
| **group_ratio**（分组纯折扣系数 + 升级门槛） | 超管 | admin `groups`「预填分组」页增「折扣等级」Tab（或 `billing-rules` 分组区收窄语义） | 系数表 + 升级门槛配置 |
| **利润分析**（Log 聚合：模型/供应商/分组） | 超管 | admin 新增「利润分析」全新页（运营组） | KPI + 多维图表 + 亏损识别明细 |
| **超管日志全链**（C→A→B+成本+售价+利润） | 超管 | 复用 admin `logs`，增「计费视图」列与展开 | 现有日志表扩列 + 行展开抽屉 |
| **C→A 客户映射**（自助别名，开放输入+候选下拉） | 客户 | console 新增「模型映射」全新页（接入组） | 别名表 + 候选下拉输入（本文重点） |
| **key 级自我约束**（可选限定模型/端点，默认全开） | 客户 | 复用 console `keys` 建/改 key 抽屉，增「使用范围」区块 | 抽屉内可选折叠区，强调"减法省心" |
| **客户日志/用量**（只显 C→A，无 B 无成本） | 客户 | 复用 console `usage` / `logs`，确保字段裁剪 | 现有视图保证不泄露 B/成本 |
| **协议兼容呈现**（一个 key 两协议都能用） | 客户 | 复用 console `keys` key 详情抽屉 + api-docs | key 详情「接入信息」区块 + 文档提示 |
| **公开站基准价/品质三价/登录享折扣** | 访客 | 复用 web-public `models` / `pricing` 增强 | 品质分级卡 + 折扣提示带，不暴露成本/B |

**贯穿三道闸（B 客户不可见）落到交互层**：
1. console 与 web-public 的任何模型选择/联想/日志/用量视图，**数据源只到 A（PublicModel 上架并集）**，永不返回 B。
2. 客户日志/用量列里**不存在成本/利润/真实模型列**；这些列只在 admin 视图出现。
3. C→A 候选下拉的候选来源 = **平台公开模型 A 全集**（PublicModel.Enabled=true 的并集），不含 B。

# A. 超管 admin 侧

## A1. 对外模型管理（PublicModel + A→B 映射 + 品质拆分）
**落点**：在现有 `admin/models-admin.html` 的「对外模型」Tab **增强**（不新建页）。现页已有 Tab 框架（对外模型 / 供应商元数据），本设计把「对外模型」Tab 的列与抽屉补齐为商品目录权威。

### A1.1 对外模型列表（Tab 主区）
- **用途**：超管管理对客户售卖的全部对外模型 A 的商品目录（建/改/上下架/品质标签/基准售价）。
- **列字段**（对齐 PublicModel）：
  - 对外名 `PublicName`(A，mono)｜展示名 `DisplayName`｜品质标签 `QualityTier`（徽章：满血 full / 增强 max / 经济 air / 自定义）｜基准售价 `BasePriceRatio`（输入×输出倍率，mono）｜底仓映射 A→B（显示当前映射的 B，**仅超管可见**）｜兜底渠道数（挂在该 B 下的同品质渠道计数，点击跳 channels）｜状态 `Enabled`（上架/下架开关）｜排序 `SortOrder`｜操作（编辑/拆品质/上下架/删除）。
- **工具条**：搜索（对外名/展示名）｜品质筛选｜状态筛选（全部/上架/下架）｜「+ 新建对外模型」｜「品质拆分引导」入口。
- **状态**：上架（绿点）/ 下架（灰点）/ 成本缺失告警（若该 A→B 下有渠道未配 ChannelModelCost，行尾挂浅黄"成本未配"小标，提示去供应商成本 Tab 补，呼应架构"成本缺失不阻断但告警"）。

### A1.2 新建/编辑对外模型（抽屉，复用 keys 抽屉范式）
分三段，自上而下：
1. **商品信息**：对外名 A（mono，唯一，建后只读）｜展示名｜品质标签下拉（full/max/air/自定义）｜描述｜排序｜上架开关。
2. **基准售价**（DECISIONS §4 售价挂 A）：输入倍率 / 输出倍率（mono 数字输入）。保存时按架构 ADR-BILL-01a 同步刷 `model_ratio` KV——交互上无感，提示语"保存后即时生效，对所有客户恒定，不随内部供应商切换波动"。
3. **底仓映射 A→B**（PlatformModelMapping，客户不可见）：
   - 单选「真实上游模型 B」（开放输入 + 候选下拉，候选来源 = 已配渠道支持的上游模型名）。
   - B 选定后，**只读展示「兜底池」**：该 (group × B) 下当前挂载的同品质渠道列表（渠道名 / 优先级 / 权重 / 该渠道对 B 的成本倍率，若缺则标"未配成本"）。这是 channels 路由的镜像视图，**不在此编辑路由**，提供「去渠道管理调整 →」跳转。
   - 强红线提示（DECISIONS §3）："此处只挂**同品质**供应商；品质不同请用『品质拆分』拆成独立对外模型，切勿混入兜底池。"

### A1.3 品质拆分引导（DECISIONS §3 关键决策）
- **用途**：把一个 opus 引导拆成 满血/增强/经济（air）三条独立 PublicModel、三价、各映射对应品质 B。
- **交互**：列表工具条「品质拆分引导」或某行操作「拆品质」打开一个**引导式弹窗**：
  - 第 1 步：选基底模型族名（如 `opus-4.8`）。
  - 第 2 步：勾选要生成的品质档（满血 / 增强 max / 经济 air，可增删自定义档），系统按命名约定预填对外名（`opus-4.8` / `opus-4.8-max` / `opus-4.8-air`）。
  - 第 3 步：每档分别填 基准售价 + 选 真实模型 B（强制每档 B 不同、且同档内渠道同品质）。
  - 第 4 步：确认 → 一次性生成三条 PublicModel + 三条 A→B 映射。
- **防呆**：若两档选了同一个 B，红色拦截提示"不同品质必须映射不同上游"。

## A2. 供应商成本配置（ChannelModelCost，手填倍率）
**落点**：在 `admin/models-admin.html` **新增一个 Tab「供应商成本」**（与现有「对外模型」「供应商元数据」并列）。不新建独立页——成本与模型同属"模型/供应商"信息域。

- **用途**：超管手填每个「渠道 × 真实模型 B」的成本倍率（DECISIONS §4「A 起步手填」）。
- **呈现（二维成本表）**：
  - 行 = 真实模型 B；可展开/分组显示挂在该 B 下的各渠道（供应商）。
  - 列：真实模型 B(mono)｜渠道/供应商名｜成本倍率 `CostRatio`（输入，mono，行内可编辑）｜补全成本倍率 `CompletionCostRatio`（可选）｜生效时间 `EffectiveTime`｜启用 `Enabled`｜「关联对外模型」（反查哪些 A 映射到此 B，便于理解影响面）。
  - **同一对外模型挂多供应商各自成本不同**的呈现：按 B 分组，组内每渠道一行，并排显示各家 CostRatio（如 X=0.40 / Y=0.55），一眼看出进货价差异。
- **工具条**：按渠道筛选｜按真实模型筛选｜「未配成本」快捷筛选（找出有渠道挂载却没填成本的 B，补全告警）｜批量"标记生效"。
- **状态**：已配（正常）/ 未配（浅黄"成本缺失"，提示不阻断计费但利润算不清）/ 已停用。
- **录入方式说明条**：页顶提示"本期为手填倍率（成本=tokens×倍率，不乘分组折扣）；自动折算进货单价为后续能力"。

## A3. 多供应商兜底（衔接现有渠道路由，不重做）
**落点**：**复用** `admin/channels.html` 现有渠道路由界面（优先级/权重/分组）；**不新增路由页**。
- 对外模型抽屉 A1.2 的「兜底池」是只读镜像，编辑入口统一指向 channels。
- 在 channels 渠道编辑里（现有界面），对每个渠道的"支持模型"即对齐 Ability 的 `(group × B)`。本设计仅要求：渠道编辑保存后，对外模型抽屉的兜底池视图能反映最新优先级/权重/兜底顺序。
- **红线落到交互**：channels 给渠道挂模型 B 时，提示"同一 B 下渠道应同品质"；品质差异请走对外模型拆分（A1.3）。

## A4. 分组折扣管理（group_ratio，纯折扣等级）
**落点**：在 `admin/groups.html`（预填分组页）**新增「折扣等级」Tab**；或在 `admin/billing-rules.html` 分组倍率区**语义收窄**为纯折扣（二选一，推荐放 groups，因其本就是"分组"信息域）。本设计取 groups 增 Tab。
- **用途**：配分组折扣系数 + 升级门槛；**强调不再控模型权限**（DECISIONS §5 §6）。
- **字段/交互**：
  - 折扣系数表：分组名（free/vip/svip/自定义）｜折扣系数（如 free=1.0 / vip=0.85 / svip=0.7，mono 行内编辑，落 `group_ratio` KV）｜适用客户数（统计）｜状态。
  - 升级门槛：每档配「按累计充值额 / 付费包等级自动升档」阈值（复用 Subscription UpgradeGroup 语义）。
- **关键提示条（消除旧认知）**：醒目说明"分组**只决定折扣**，不再决定能用哪些模型——所有上架对外模型对所有分组全开。"（DECISIONS §5）
- **下线项**：移除/隐藏旧"分组圈定可用模型"配置（GroupSpecialUsableGroup 圈模型用途停用），避免与"全开"矛盾。

## A5. 利润分析看板（Log 聚合，DECISIONS §9）
**落点**：**全新页** `admin/profit.html`，挂在 admin 侧栏「运营」组（与 计费规则/任务监控 并列）。新增导航项需在 admin-shell NAV 的「运营」组加一条（仅此一处导航增量）。
- **用途**：识别"哪个供应商在亏、该给谁导流、哪个对外模型定价偏低、哪个等级折扣过狠"。
- **顶部 KPI 带**：总售价 `Σquota_sell`｜总成本 `Σquota_cost`｜总利润 `Σquota_profit`｜整体利润率 `Σprofit/Σsell`｜时间范围切换（今日/7日/30日/自定义）。
- **三个分析维度（Tab 或并排卡）**：
  1. **按对外模型 A**（`GROUP BY resolved_public_model`）：每模型 售价/成本/利润/利润率 排序表 + 利润率分布图；**红色高亮利润率低/为负的模型**（→ 定价偏低信号）。
  2. **按供应商/渠道**（`GROUP BY channel_id` join 渠道名）：每供应商 成本/利润 + **亏损供应商置顶红标**（→ 该减量或调成本）；并给"建议导流"提示（同 B 下哪个渠道利润更高 → 提权重）。
  3. **按分组**（`GROUP BY group`）：每等级 售价/利润/利润率 → 看哪个折扣档过狠（svip 利润最薄是预期，但若为负则告警）。
- **明细钻取**：任一聚合行可下钻到对应 Log 明细（跳 admin/logs 并带筛选条件）。
- **数据来源说明**：本期为 Log 表实时聚合（架构 ADR-BILL-07），无独立数仓；大时间范围查询走 created_at/channel/model/group 索引。

## A6. 超管日志全链视图（DECISIONS §8）
**落点**：**复用** `admin/logs.html`，**增「计费/兼容」列与行展开**（不新建页）。
- **列扩展**：在现有日志列后增 请求模型 C｜对外模型 A｜真实模型 B｜实际供应商（渠道名）｜入站协议｜上游协议｜协议转换标记｜售价 `quota_sell`｜成本 `quota_cost`｜利润 `quota_profit`。列多 → 默认精简，「列设置」可勾选展开（mono 金额列）。
- **行展开/详情抽屉**：完整三段式链路 `C → A → B`（带映射来源层标注 用户级/分组级/超管层）+ 实际渠道 + 协议头尾对比 + 售价/成本/利润分解。
- **筛选**：按对外模型 A / 真实模型 B / 供应商 / 分组 / 协议转换与否 / 利润为负。
- **权限**：成本/利润/真实模型 B 列**仅 admin/root 可见**（对齐序列化层 AdminLogView）；与 console 客户日志严格隔离。

# B. 用户端 console 侧（极简体验）

## B1. 我的模型映射（C→A 客户自助别名表）★本文重点★
**落点**：**全新页** `console/model-map.html`（别名"模型映射"），挂在 console 侧栏「接入」组（与 API密钥/用量统计/异步任务 并列）。需在 console-shell NAV「接入」组加一条（仅此一处导航增量）。

- **用途**：客户把自己应用里写死的 model 名（C）映射到平台公开模型（A），客户端零改动接入。**作用域：分组级 / 用户级，用户级优先于分组级**（DECISIONS §2）。
- **页面结构**：
  - 顶部说明条："把你应用里的模型名（如 `gpt-4o`）映射到平台模型，无需改代码即可切换底层。映射只是改名，**所有平台模型对你全开**。"
  - 作用域切换：「用户级映射」/「分组级映射」两段（用户级条目带"覆盖分组级"标识；同名时用户级生效）。
  - 别名映射表：列 = 你的模型名 C（mono）｜→｜平台模型 A（mono + 候选状态）｜来源（用户级/分组级，分组级行在用户级视图下为只读继承）｜状态（已生效/平台无此模型告警）｜操作（编辑/删除）。
  - 「+ 添加映射」按钮。

### B1.1 客户自助模型映射（C→A）候选下拉交互【重点设计】
这是本功能的核心交互，严格落 DECISIONS §2「开放输入 + 候选提示，不强制白名单」与三道闸「候选层只返平台公开模型 A 全集」：

- **左侧「你的模型名 C」输入框**：纯自由文本（mono 占位 `例如 gpt-4o`）。客户填什么都行——这是他应用里现有的名字，平台不约束。
- **右侧「平台模型 A」输入框 = 开放输入 + 候选下拉（核心）**：
  - **开放输入**：是一个可自由键入的文本框，不是封闭 select。客户可以输入任意字符串。
  - **联想候选**：键入时实时下拉候选列表，**候选来源 = 平台公开模型 A 全集（PublicModel.Enabled=true 的并集，绝不含真实模型 B）**。例如输入 `gpt` 联想出 `gpt-4o`、`gpt-4o-mini`…；输入 `opus` 联想出 `opus-4.8`、`opus-4.8-max`、`opus-4.8-air` 三个品质档（呼应品质拆分，让客户看见三个独立可选项与各自价格）。
  - **候选项内容**：每个候选显示 对外名 A（mono）+ 展示名 + 品质标签徽章 + 该客户折后单价（按其分组系数预算，帮客户选型）。**不显示成本、不显示真实模型 B。**
  - **可点选**：点候选即填入，标记为"已匹配平台模型"（绿色对勾），状态=已生效。
  - **允许自由输入平台没有的名字**：客户硬输一个候选里没有的字符串（如平台无 `my-custom-x`），**不拦截、允许保存**；但输入框右侧给**浅黄非阻断提示**："该名称不在平台模型列表中，调用时可能失败。"（DECISIONS §2「可给浅黄提示但不拦」）。状态标"平台未收录"。
  - **键盘可达**：↑↓ 选候选、Enter 确认、Esc 关下拉；候选高亮匹配子串。
  - **空候选态**：输入无任何匹配时，下拉显示"无匹配的平台模型，可直接保存为自定义名（调用时可能失败）"，引导但不阻断。
- **作用域选择**：每条映射保存时选「仅本人（用户级）」或「整个分组（分组级，需分组管理权限/由超管侧下发，客户视图通常只读分组级、可编辑用户级）」。用户级优先级高，UI 在冲突时提示"此映射将覆盖分组级的同名映射"。
- **校验/防呆**：左侧 C 不可为空；同一作用域内 C 唯一（重复时提示覆盖）；不做 A 白名单强校验（自由输入是产品决策，不是 bug）。
- **为何不强制白名单（产品说明，留给文案）**：模型本就全开 + B 不可见，客户自由映射安全（DECISIONS §5）；强校验反而挡住客户合理的"先占位、平台后续上架"用法。

## B2. 建 key 时自我约束（key 级减法，非权限）
**落点**：**复用** `console/keys.html` 现有「建/改 key 抽屉」，**增一个可选折叠区「使用范围（可选）」**（默认折叠、默认全开）。
- **用途**：客户给自己上保险——限定某 key 只能用某些模型/某些端点（防泄露爆量、限制某项目只用便宜模型）。DECISIONS §7 强调**减法省心、非权限闸门**。
- **交互**：
  - 默认：折叠区收起，提示"默认可用全部模型与端点"。
  - 展开后两块：
    1. **限定模型（可选）**：多选 chips，候选来源 = 平台公开模型 A 全集（同 B1.1 候选源，含品质档）。默认不选 = 全部。选中即"只允许这些"（减法）。
    2. **限定端点协议（可选）**：勾选 OpenAI 格式 / Anthropic(/v1/messages) 格式。默认全勾 = 全开（落 Token.EndpointLimits）。
  - **文案铁律**：标题/说明明确"这是**做减法的自我约束**（缩小范围、防误用），不是授权——平台模型本就对你全开，无需也无法在此**加**权限。"（DECISIONS §7）
- **字段对齐**：Token.ModelLimits（校验对象=A）+ Token.EndpointLimits；`*_enabled=false` 即默认全开不校验。

## B3. 客户日志/用量视图（只显 C→A，无 B 无成本）
**落点**：**复用** `console/usage.html`（用量统计）+ console 客户日志（如有，或在 usage 内的明细）。**不新建页**，重点是**字段裁剪正确**。
- **用途**：客户看自己每笔用了啥、花了多少（折后价）。
- **可见列**：时间｜请求模型 C｜命中的平台模型 A｜token 用量（输入/输出）｜本笔花费（**折后价 = 基准价×自己分组系数**，mono）｜所用 key。
- **绝不出现**：真实模型 B、成本、利润、实际供应商渠道、上游协议（这些是 admin-only）。对齐三道闸序列化层 UserLogView。
- **聚合视图**：按 C 或按 A 看消耗 Top、按 key 看消耗、按日趋势——全部只到 A 维度。
- **折扣呈现**：用量/账单里显示"已享 vip 折扣 0.85"之类的等级标识，让客户感知折扣价值（但只展示折后结果与折扣率，不展示成本）。

## B4. 协议兼容呈现（一个 key 两协议都能用）
**落点**：**复用** `console/keys.html` 的 **key 详情抽屉**增「接入信息」区块 + `api-docs`（quickstart/authentication）文档提示。不新建页。
- **用途**：让客户明确知道"同一个 key，OpenAI 格式和 Anthropic(/v1/messages) 格式都能打"（DECISIONS §1）。
- **key 详情「接入信息」区块**：
  - 展示 `base_url`（统一网关地址）+ 该 key（脱敏可复制）。
  - 「支持端点」列表：OpenAI 兼容（`/v1/chat/completions` 等）+ Anthropic 兼容（`/v1/messages`），各给一键复制的 curl 示例与"同一 key 通用"标注。
  - 若该 key 在 B2 限定了端点，则此处只显示其放行的端点，并标"本 key 已限定为 …"。
- **文档侧**：api-docs quickstart/authentication 增一段"一个 key 两种协议"说明，呼应 key 详情。

# C. 公开站侧（web-public，不暴露成本/真实模型 B）

## C1. 模型广场 / 价格页（基准价 + 品质三价 + 登录享折扣）
**落点**：**复用并增强** `web-public/models.html`（模型广场）与 `web-public/pricing.html`（价格页）。不新建页。
- **用途**：对访客展示对外模型 A 的**基准价**，体现品质分级与"登录后按等级享折扣"，吸引注册转化。
- **数据源**：仅 PublicModel 上架记录（A + 品质标签 + 基准售价 + 展示名/描述），**绝不含 成本、真实模型 B、供应商**。
- **模型广场（models）增强**：
  - 模型卡按"模型族"聚合，**品质拆分可视化**：一个 opus 族卡内并列 满血 / 增强 max / 经济 air 三个品质档，各自标品质徽章 + 各自基准价（DECISIONS §3 三价）。
  - 卡上不出现任何内部/成本信息；筛选按 品质/能力/上下文长度（皆来自 PublicModel 展示属性）。
- **价格页（pricing）增强**：
  - 基准价表：每个对外模型 A 的 输入/输出 基准单价（由 BasePriceRatio 折算展示）。
  - 品质三价并列：opus 三档独立行/列，价不同。
  - **折扣提示带**：醒目"登录后按会员等级享折扣（vip 0.85 / svip 0.7 示例）"，配"立即注册/登录查看你的专属价"CTA；**未登录只显示基准价**，登录后（若价格页支持登录态）可叠加显示折后价。
  - **不暴露**：成本、利润、真实模型 B、各供应商——价格页只是"牌价 + 等级折扣"的对外橱窗。
