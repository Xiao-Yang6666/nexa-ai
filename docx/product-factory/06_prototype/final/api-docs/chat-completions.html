聊天补全 API

给定一组对话消息，模型返回一条补全回复。这是 Nexa·AI 最核心的接口，完全兼容 OpenAI Chat Completions 协议，支持多轮对话、函数调用、流式输出与多模态输入。

端点

POST https://api.nexa.ai/v1/chat/completions

需要在 Authorization 头携带有效的 Bearer Token，详见 认证说明。请求体为 JSON，model 与 messages 为必填字段。

请求示例

curl https://api.nexa.ai/v1/chat/completions \
  -H "Authorization: Bearer $NEXA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "system", "content": "你是一个简洁的助手。"},
      {"role": "user", "content": "什么是向量数据库？"}
    ],
    "temperature": 0.7,
    "max_tokens": 512
  }'
from openai import OpenAI

client = OpenAI(
    base_url="https://api.nexa.ai/v1",
    api_key="sk-nexa-...",
)

resp = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是一个简洁的助手。"},
        {"role": "user", "content": "什么是向量数据库？"},
    ],
    temperature=0.7,
    max_tokens=512,
)
print(resp.choices[0].message.content)
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.nexa.ai/v1",
  apiKey: process.env.NEXA_API_KEY,
});

const resp = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [
    { role: "system", content: "你是一个简洁的助手。" },
    { role: "user", content: "什么是向量数据库？" },
  ],
  temperature: 0.7,
  max_tokens: 512,
});
console.log(resp.choices[0].message.content);
package main

import (
    "context"
    "fmt"

    "github.com/sashabaranov/go-openai"
)

func main() {
    cfg := openai.DefaultConfig("sk-nexa-...")
    cfg.BaseURL = "https://api.nexa.ai/v1"
    client := openai.NewClientWithConfig(cfg)

    resp, _ := client.CreateChatCompletion(context.Background(), openai.ChatCompletionRequest{
        Model: "gpt-4o",
        Messages: []openai.ChatCompletionMessage{
            {Role: "system", Content: "你是一个简洁的助手。"},
            {Role: "user", Content: "什么是向量数据库？"},
        },
        Temperature: 0.7,
        MaxTokens: 512,
    })
    fmt.Println(resp.Choices[0].Message.Content)
}

请求参数

请求体为 JSON 对象，支持以下字段：

参数类型必填说明
modelstring必填要使用的模型 ID，如 gpt-4o、claude-3-5-sonnet、deepseek-chat。完整列表见 模型列表。
messagesarray必填对话消息数组，每个元素含 role（system / user / assistant / tool）与 content。
temperaturenumber可选采样温度，范围 0–2，默认 1。值越高输出越随机，越低越确定。
top_pnumber可选核采样阈值，范围 0–1，默认 1。与 temperature 二选一调节即可。
max_tokensinteger可选本次补全生成的最大 token 数，不含输入。
streamboolean可选为 true 时以 SSE 流式逐块返回，默认 false。
stopstring / array可选最多 4 个停止序列，模型生成到任一序列即停止。
presence_penaltynumber可选范围 -2–2，正值鼓励谈及新话题，默认 0。
frequency_penaltynumber可选范围 -2–2，正值降低重复用词的概率，默认 0。
ninteger可选为每条输入生成的补全条数，默认 1。
toolsarray可选可供模型调用的工具（函数）定义列表，用于函数调用。
tool_choicestring / object可选控制是否及如何调用工具：auto / none / 指定函数。
response_formatobject可选设为 {"type":"json_object"} 强制返回合法 JSON。
userstring可选代表终端用户的稳定标识，用于滥用监测。
跨厂商参数对齐。Nexa 会把上述统一参数自动映射到各厂商的原生字段。若某模型不支持某参数（如部分模型无 frequency_penalty），网关会安全地忽略而非报错。

响应

非流式请求返回一个 chat.completion 对象。主要字段如下：

字段类型说明
idstring本次补全的唯一标识。
objectstring固定为 chat.completion。
createdinteger创建时间的 Unix 时间戳（秒）。
modelstring实际服务本次请求的模型 ID。
choicesarray补全结果数组，含 index、message、finish_reason。
usageobjecttoken 用量：prompt_tokens、completion_tokens、total_tokens。
 
{
  "id": "chatcmpl-b3e1f9a2",
  "object": "chat.completion",
  "created": 1718857800,
  "model": "gpt-4o",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "向量数据库是一种为高维向量的相似度检索而优化的数据库。"
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 29,
    "completion_tokens": 34,
    "total_tokens": 63
  }
}

流式响应

当请求中设置 "stream": true 时，网关以 Server-Sent Events（SSE） 协议逐块推送增量。每个事件以 data: 开头，承载一个 chat.completion.chunk 对象，增量内容位于 choices[0].delta.content。流以 data: [DONE] 结束。

curl https://api.nexa.ai/v1/chat/completions \
  -H "Authorization: Bearer $NEXA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "数到三"}],
    "stream": true
  }'
stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "数到三"}],
    stream=True,
)
for chunk in stream:
    delta = chunk.choices[0].delta.content or ""
    print(delta, end="", flush=True)
const stream = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: "数到三" }],
  stream: true,
});
for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0].delta.content ?? "");
}
stream, _ := client.CreateChatCompletionStream(context.Background(), openai.ChatCompletionRequest{
    Model: "gpt-4o",
    Messages: []openai.ChatCompletionMessage{{Role: "user", Content: "数到三"}},
    Stream: true,
})
defer stream.Close()
for {
    resp, err := stream.Recv()
    if err != nil {
        break
    }
    fmt.Print(resp.Choices[0].Delta.Content)
}

SSE 数据流的原始格式如下，每个 chunk 携带一小段增量：

data: {"id":"chatcmpl-c4...","object":"chat.completion.chunk","choices":[{"delta":{"role":"assistant"},"index":0}]}

data: {"choices":[{"delta":{"content":"一"},"index":0}]}

data: {"choices":[{"delta":{"content":"、二、三"},"index":0}]}

data: {"choices":[{"delta":{},"index":0,"finish_reason":"stop"}]}

data: [DONE]
流式下没有 usage 字段。默认流式响应的 chunk 不包含 token 用量。若需统计，请在请求中加入 "stream_options": {"include_usage": true}，网关会在 [DONE] 前补发一个仅含 usage 的 chunk。
