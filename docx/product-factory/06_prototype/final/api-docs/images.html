图像 API

根据文本提示词生成图像。完全兼容 OpenAI Images 协议，支持多种尺寸、质量档位与返回格式，可在 URL 与 Base64 之间自由选择，便于直接落库或前端渲染。

端点

POST https://api.nexa.ai/v1/images/generations

需要在 Authorization 头携带有效的 Bearer Token，详见 认证说明。请求体为 JSON，model 与 prompt 为必填字段。生成图像的计费按张数与尺寸/质量档位累计。

请求示例

curl https://api.nexa.ai/v1/images/generations \
  -H "Authorization: Bearer $NEXA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "dall-e-3",
    "prompt": "薄雾笼罩的青色山谷，等距视角，柔光",
    "n": 1,
    "size": "1024x1024",
    "quality": "hd",
    "response_format": "url"
  }'
from openai import OpenAI

client = OpenAI(
    base_url="https://api.nexa.ai/v1",
    api_key="sk-nexa-...",
)

resp = client.images.generate(
    model="dall-e-3",
    prompt="薄雾笼罩的青色山谷，等距视角，柔光",
    size="1024x1024",
    quality="hd",
)
print(resp.data[0].url)
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.nexa.ai/v1",
  apiKey: process.env.NEXA_API_KEY,
});

const resp = await client.images.generate({
  model: "dall-e-3",
  prompt: "薄雾笼罩的青色山谷，等距视角，柔光",
  size: "1024x1024",
  quality: "hd",
});
console.log(resp.data[0].url);
package main

import (
    "context"
    "fmt"

    "github.com/sashabaranov/go-openai"
)

func main() {
    cfg := openai.DefaultConfig("sk-nexa-...")
    cfg.BaseURL = "https://api.nexa.ai/v1"
    client := openai.NewClientWithConfig(cfg)

    resp, _ := client.CreateImage(context.Background(), openai.ImageRequest{
        Model: openai.CreateImageModelDallE3,
        Prompt: "薄雾笼罩的青色山谷，等距视角，柔光",
        Size: openai.CreateImageSize1024x1024,
        N: 1,
    })
    fmt.Println(resp.Data[0].URL)
}

请求参数

请求体为 JSON 对象，支持以下字段：

参数类型必填说明
modelstring必填图像模型 ID，如 dall-e-3、flux-pro、sd3-large。
promptstring必填描述目标图像的文本提示词。dall-e-3 上限约 4000 字符，越具体越可控。
ninteger可选生成图像张数，默认 1。dall-e-3 仅支持 1，其余模型上限 10。
sizestring可选输出尺寸：1024x1024（默认）/ 1792x1024 / 1024x1792。
qualitystring可选质量档位：standard（默认）或 hd（更精细，耗时与计费更高）。
response_formatstring可选返回形式：url（默认，临时链接，约 1 小时有效）或 b64_json（内联 Base64）。
stylestring可选vivid（默认，更具张力）或 natural（更写实），仅 dall-e-3 支持。
userstring可选代表终端用户的稳定标识，用于滥用监测。
URL 是临时链接。默认返回的 url 有效期约 1 小时，请及时下载并落库到你自己的对象存储。若希望直接拿到字节流，把 response_format 设为 b64_json 即可在响应里取得 Base64 内容。

响应字段

返回对象包含生成时间戳与图像数组。每个元素含图像数据及（部分模型的）修订后提示词。

字段类型说明
createdinteger生成时间的 Unix 时间戳（秒）。
dataarray图像对象数组，长度等于 n。
urlstringresponse_format=url 时返回的临时图像链接。
b64_jsonstringresponse_format=b64_json 时返回的 Base64 图像内容。
revised_promptstring模型实际采用的优化后提示词（dall-e-3 会自动改写）。
 
{
  "created": 1718861400,
  "data": [{
    "url": "https://cdn.nexa.ai/img/8f3a1c2e.png",
    "revised_prompt": "等距视角下薄雾缭绕的青绿山谷，柔和的清晨侧光。"
  }]
}

图像模型

Nexa 接入多家厂商的图像生成模型，统一在同一接口下暴露。常用模型如下：

模型 ID厂商最大尺寸特点
dall-e-3OpenAI1792x1024提示词遵循度高，自动改写优化。
dall-e-2OpenAI1024x1024速度快，支持单次多张。
flux-proBlack Forest Labs1440x1440细节锐利，写实质感强。
flux-schnellBlack Forest Labs1024x1024极速出图，适合预览迭代。
sd3-largeStability AI1536x1536排版与文字渲染表现优秀。
内容合规由网关统一审查。提示词与生成结果会经过安全策略过滤，违规请求将返回 400 并附带 content_policy_violation 错误码，详见 错误码参考。
