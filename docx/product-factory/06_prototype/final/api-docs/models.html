模型列表 API

返回当前 API Key 可访问的全部模型及其元信息。用于在运行时动态发现可用模型、构建模型选择器，或校验某个模型 ID 是否有效。完全兼容 OpenAI Models 接口。

端点

GET https://api.nexa.ai/v1/models

无需请求体，仅需在 Authorization 头携带有效 Bearer Token。返回结果会根据该 Key 的权限范围自动过滤——若 Key 仅被授权 gpt-*，则列表只含 OpenAI 系模型。

请求示例

curl https://api.nexa.ai/v1/models \
  -H "Authorization: Bearer $NEXA_API_KEY"
from openai import OpenAI

client = OpenAI(
    base_url="https://api.nexa.ai/v1",
    api_key="sk-nexa-...",
)

for m in client.models.list():
    print(m.id, m.owned_by)
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.nexa.ai/v1",
  apiKey: process.env.NEXA_API_KEY,
});

const list = await client.models.list();
for (const m of list.data) console.log(m.id, m.owned_by);
package main

import (
    "context"
    "fmt"

    "github.com/sashabaranov/go-openai"
)

func main() {
    cfg := openai.DefaultConfig("sk-nexa-...")
    cfg.BaseURL = "https://api.nexa.ai/v1"
    client := openai.NewClientWithConfig(cfg)

    list, _ := client.ListModels(context.Background())
    for _, m := range list.Models {
        fmt.Println(m.ID, m.OwnedBy)
    }
}

响应字段

返回一个 list 对象，data 数组中每个元素是一个 model 对象。

字段类型说明
objectstring顶层固定为 list；每个元素固定为 model。
dataarray模型对象数组。
idstring模型唯一标识，即调用时传入的 model 值。
createdinteger模型上线时间的 Unix 时间戳（秒）。
owned_bystring模型所属厂商，如 openai、anthropic、deepseek。

  "object": "list",
  "data": [

      "object": "model",
      "created": 1715367049,
      "owned_by": "openai"
    },
    {

      "owned_by": "anthropic"
    },
    {
      "id": "deepseek-chat",
      "object": "model",
      "created": 1716508800,
      "owned_by": "deepseek"
    }
  ]
}

支持的模型族

Nexa 持续接入主流厂商的最新模型，统一在同一接口下暴露。当前支持的主要模型族：

厂商模型族示例能力
OpenAIgpt-4o · gpt-4o-mini · o1对话 · 函数调用 · 视觉 · 推理
Anthropicclaude-3-5-sonnet · claude-3-haiku对话 · 长上下文 · 视觉
Googlegemini-1.5-pro · gemini-1.5-flash对话 · 超长上下文 · 多模态
DeepSeekdeepseek-chat · deepseek-reasoner对话 · 推理 · 代码
Qwenqwen-max · qwen-plus对话 · 中文优化 · 长上下文
Mistralmistral-large · mistral-small对话 · 函数调用
动态发现优于硬编码。新模型会随厂商发布持续上线。建议在应用中通过本接口动态拉取可用模型列表，而非把模型 ID 写死在代码里，以便自动获得最新能力。完整定价见 模型广场。
