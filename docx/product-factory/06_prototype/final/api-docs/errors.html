错误码参考

Nexa·AI 沿用标准 HTTP 状态码语义，并在响应体中返回结构化的错误对象。本页列出全部状态码、业务错误码及其常见原因与处理建议，帮助你在客户端实现健壮的错误处理。

概览

所有错误响应均以非 2xx 的 HTTP 状态码返回，响应体为统一的 JSON 结构。状态码大类含义如下：2xx 成功；4xx 客户端错误（请求本身有问题，重试通常无效，需修正后再发）；5xx 服务端错误（网关或上游临时故障，可按退避策略重试）。

错误结构

错误响应体始终包含一个 error 对象，字段含义：message 人类可读的描述；type 错误大类；code 机器可读的业务错误码；param 触发错误的具体参数（若适用）。

{
  "error": {
    "message": "You exceeded your current quota, please check your plan and billing details.",
    "type": "insufficient_quota",
    "code": "insufficient_quota",
    "param": null
  }
}

HTTP 状态码

下表覆盖网关可能返回的全部 HTTP 状态码及其典型语义：

状态含义常见原因处理建议
400Bad Request请求体非法、缺必填字段、参数取值越界。检查 error.param 指出的字段并修正后重发。
401Unauthorized缺少 / 无效 / 已撤销的 API Key。核对 Authorization 头与 Key 是否仍有效。
403ForbiddenKey 无该模型 / 该接口的访问权限。在控制台为分组授予对应模型权限。
404Not Found端点路径错误，或模型 ID 不存在。核对 URL 与 model，参见 模型列表。
429Too Many Requests触发 RPM / TPM 限流，或额度已用尽。读取 Retry-After 退避重试，见 限流说明。
500Internal Server Error网关内部异常。指数退避重试；持续出现请联系支持。
502Bad Gateway上游模型供应商返回了无效响应。稍后退避重试，或切换备用模型。
503Service Unavailable上游过载或维护中，暂时不可用。按 Retry-After 退避重试。

业务错误码

error.code 提供比 HTTP 状态码更细的机器可读分类，便于在代码中做精准分支处理：

状态error.code含义处理建议
401invalid_api_key提供的 API Key 无效或格式错误。重新生成 Key 并更新环境变量。
403model_access_denied当前 Key 未被授权访问该模型。在分组权限中开启该模型。
404model_not_found请求的 model 不存在或已下线。改用 模型列表 中的有效 ID。
400invalid_request_error请求结构或字段取值不合法。依据 message 与 param 修正。
400context_length_exceeded输入 token 超出模型上下文窗口。缩短输入或改用更大上下文的模型。
400content_policy_violation提示词或输出触发内容安全策略。调整内容后重试。
429rate_limit_exceeded超出每分钟请求或 token 速率上限。按 Retry-After 退避，降低并发。
429insufficient_quota账户额度已用尽或欠费。前往控制台充值或调整套餐。
502upstream_error上游供应商返回错误。退避重试，或在请求中指定备用模型。
503engine_overloaded上游引擎当前过载。稍后重试或降低请求频率。

处理建议

4xx 不要盲目重试。除 429 外的客户端错误（400/401/403/404）通常源于请求本身的问题，原样重发只会再次失败。应先依据 error.code 与 message 修正请求，再重新提交。
5xx 与 429 才适合重试。对 429/500/502/503 采用指数退避（含抖动）重试，并尊重响应中的 Retry-After 头；设置最大重试次数以避免雪崩。具体退避策略与示例代码见 限流说明。
始终以 error.code 为分支依据。message 文案可能随版本调整，而 code 是稳定契约。在代码中按 code 做精确匹配，能让错误处理在网关升级后依然可靠。
