更新日志

记录 Nexa·AI 网关的模型接入、能力增强与问题修复。我们遵循向后兼容原则——除非另行公告，既有接口与字段保持稳定。下方按时间倒序排列各版本。

图例

每条变更以标签标注类型：

  • 新增引入全新的模型、端点或能力。
  • 优化对既有能力的性能、体验或行为改进。
  • 修复修正缺陷或不符合预期的行为。

版本时间线

  • v2.8.02026-06-12
    • 新增接入 gemini-2.0-flash 与 gemini-2.0-pro，支持百万级长上下文。
    • 优化流式响应首 token 延迟（TTFT）平均降低约 22%。
  • v2.7.22026-05-28
    • 修复修复 embeddings 接口在传入 dimensions 时偶发维度未截断的问题。
    • 修复修正部分 429 响应缺失 Retry-After 头的情况。
  • v2.7.02026-05-15
    • 新增图像接口接入 flux-pro 与 flux-schnell，支持更高分辨率出图。
    • 新增聊天补全支持 stream_options.include_usage，可在流式末尾返回 token 用量。
    • 优化统一各厂商的 finish_reason 取值，行为与 OpenAI 对齐。
  • v2.6.12026-04-30
    • 修复修复函数调用场景下 tool_calls 在跨厂商模型间字段映射不一致的问题。
    • 优化错误响应统一补齐 error.code 字段，便于客户端做分支处理。
  • v2.6.02026-04-18
    • 新增接入 claude-3-7-sonnet，强化代码与推理能力。
    • 新增限流响应回携 x-ratelimit-* 系列头，实时暴露配额余量。
  • v2.5.32026-03-29
    • 修复修复长上下文请求在上游超时后未正确返回 504 的问题。
    • 优化降低冷启动场景下的连接建立耗时。
  • v2.5.02026-03-10
    • 新增嵌入接口接入 bge-m3 与 gte-large-zh，强化中文检索。
    • 新增支持按分组（Group）粒度配置独立 RPM / TPM 限额。
    • 优化模型列表接口按 Key 权限自动过滤返回结果。
  • v2.4.12026-02-22
    • 修复修复 response_format=json_object 在个别模型下偶发返回非法 JSON 的问题。
  • v2.4.02026-02-05
    • 新增接入 deepseek-reasoner，支持显式推理链输出。
    • 优化聊天补全的视觉输入（image_url）在多厂商间统一编解码。
  • v2.3.02026-01-16
    • 新增图像接口接入 sd3-large，文字渲染表现显著提升。
    • 优化统一计费口径，usage 字段含上游真实 token 计数。
    • 修复修复 base64 图像返回时偶发的内容截断。
  • v2.2.02025-12-20
    • 新增正式发布 OpenAI 兼容网关，支持聊天、嵌入、图像三大接口。
    • 新增提供统一鉴权、分组管理与用量看板。
