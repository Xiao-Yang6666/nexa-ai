嵌入 API

将文本转换为高维浮点向量，用于语义检索、聚类、推荐与 RAG 召回。完全兼容 OpenAI Embeddings 协议，单次请求可批量编码多段文本，并支持指定输出维度以适配你的向量库。

端点

POST https://api.nexa.ai/v1/embeddings

需要在 Authorization 头携带有效的 Bearer Token，详见 认证说明。请求体为 JSON，model 与 input 为必填字段。input 既可以是单条字符串，也可以是字符串数组以批量编码。

请求示例

curl https://api.nexa.ai/v1/embeddings \
  -H "Authorization: Bearer $NEXA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "text-embedding-3-small",
    "input": "向量数据库用于相似度检索。",
    "encoding_format": "float"
  }'
from openai import OpenAI

client = OpenAI(
    base_url="https://api.nexa.ai/v1",
    api_key="sk-nexa-...",
)

resp = client.embeddings.create(
    model="text-embedding-3-small",
    input="向量数据库用于相似度检索。",
)
print(resp.data[0].embedding[:5])
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.nexa.ai/v1",
  apiKey: process.env.NEXA_API_KEY,
});

const resp = await client.embeddings.create({
  model: "text-embedding-3-small",
  input: "向量数据库用于相似度检索。",
});
console.log(resp.data[0].embedding.slice(0, 5));
package main

import (
    "context"
    "fmt"

    "github.com/sashabaranov/go-openai"
)

func main() {
    cfg := openai.DefaultConfig("sk-nexa-...")
    cfg.BaseURL = "https://api.nexa.ai/v1"
    client := openai.NewClientWithConfig(cfg)

    resp, _ := client.CreateEmbeddings(context.Background(), openai.EmbeddingRequest{
        Model: openai.SmallEmbedding3,
        Input: []string{"向量数据库用于相似度检索。"},
    })
    fmt.Println(resp.Data[0].Embedding[:5])
}

请求参数

请求体为 JSON 对象，支持以下字段：

参数类型必填说明
modelstring必填嵌入模型 ID，如 text-embedding-3-small、text-embedding-3-large。
inputstring / array必填待编码的文本。可为单条字符串，或字符串数组以一次性批量编码（单次上限 2048 条）。
encoding_formatstring可选向量编码格式：float（默认，浮点数组）或 base64（更紧凑，适合大批量传输）。
dimensionsinteger可选截断输出向量到指定维度，仅 text-embedding-3-* 系列支持。便于在精度与存储成本间权衡。
userstring可选代表终端用户的稳定标识，用于滥用监测。
批量编码更省成本。把多段文本放进 input 数组一次性提交，比逐条调用显著降低往返开销与计费 token。返回的 data 数组顺序与输入严格一致，可按 index 对齐。

响应字段

返回一个 list 对象，data 数组中每个元素是一个 embedding 对象。

字段类型说明
objectstring顶层固定为 list；每个元素固定为 embedding。
dataarray嵌入对象数组，顺序与输入一致。
embeddingarray浮点数向量本体；base64 格式时为编码字符串。
indexinteger该向量对应输入数组中的下标。
modelstring实际服务本次请求的嵌入模型 ID。
usageobjecttoken 用量：prompt_tokens 与 total_tokens。
 
{
  "object": "list",
  "data": [{
    "object": "embedding",
    "index": 0,
    "embedding": [-0.0123, 0.0461, -0.0078, 0.0319, 0.0205]
  }],
  "model": "text-embedding-3-small",
  "usage": {
    "prompt_tokens": 12,
    "total_tokens": 12
  }
}

嵌入模型

Nexa 接入多家厂商的嵌入模型，统一在同一接口下暴露。常用模型如下：

模型 ID默认维度可调维度适用场景
text-embedding-3-small1536支持通用语义检索，性价比首选。
text-embedding-3-large3072支持高精度召回，跨语言检索。
text-embedding-ada-0021536不支持兼容旧索引，平滑迁移。
bge-m31024不支持中文优化，稠密+稀疏混合检索。
gte-large-zh1024不支持中文通用，长文本友好。
同库内务必使用同一模型与维度。不同模型产出的向量空间不可互通；若用 dimensions 截断，索引与查询必须采用相同维度，否则相似度计算结果无意义。
