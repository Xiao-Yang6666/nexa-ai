SDK 下载

Nexa·AI 完全兼容 OpenAI 的 HTTP 协议，因此无需任何专用 SDK——直接使用各语言的 OpenAI 官方客户端，只把 base_url 指向 Nexa 即可。下面汇总各语言的安装方式与文档入口。

兼容说明

所有官方 OpenAI SDK 都暴露 base_url（或 baseURL / BaseURL）配置项。把它改为 https://api.nexa.ai/v1，并填入你的 Nexa API Key，原有代码几乎无需改动即可跨厂商调用全部模型。

唯一要改的就是两行配置。把 base_url 设为 https://api.nexa.ai/v1、api_key 设为你的 sk-nexa-... Key，其余调用方式与官方完全一致。详见 快速开始。

官方 SDK

Python
openai

OpenAI 官方 Python 客户端，同步/异步均支持，自动重试与流式解析开箱即用。

pip install openai
查看文档
Node.js
openai

官方 TypeScript/JavaScript 客户端，类型完备，支持流式与浏览器/边缘运行时。

npm install openai
查看文档
Go
sashabaranov/go-openai

社区维护、事实标准的 Go 客户端，通过 DefaultConfig 改 BaseURL 即可接入。

go get github.com/sashabaranov/go-openai
查看文档
Java
com.openai:openai-java

OpenAI 官方 Java 客户端，支持 Maven / Gradle，可通过 builder 指定 baseUrl。

implementation "com.openai:openai-java:0.20.0"
查看文档

社区 SDK

除官方客户端外，以下社区项目也已验证与 Nexa 兼容，适合更多语言与框架场景：

语言 / 框架包名说明
Rustasync-openai异步优先，基于 tokio，支持流式与重试。
PHPopenai-php/clientLaravel 友好，链式 API。
Rubyruby-openai简洁易用，含 Rails 集成示例。
.NETOpenAI-DotNet覆盖全部端点，支持 DI 注入。
Dart / Flutterdart_openai移动端可用，含流式封装。
社区 SDK 非官方维护。其兼容性与更新节奏由各自社区负责。生产环境建议优先选用官方客户端；使用社区 SDK 时请锁定版本并自行评估其错误处理是否覆盖 错误码参考 中的各类情形。
